Reference

Configuration keys, filename normalisation rules, supported formats, metadata sources, processing log concepts, selected HTTP API routes, and browser-side UI state. For the canonical technical spec, the repository README.md and source remain authoritative.

Configuration file (config.json)

Start from config.json.example. Typical keys include source_dir, destination_dir, extract_profile (flac, mp3, or aac/m4a), platinum_notes_app, pn_output_suffix, target_lufs, and target_true_peak. Positive LUFS input (e.g. 11.5) is interpreted as negative integrated loudness where the app supports that shorthand.

Filename search and tags

The app does not search on raw stems. It normalises DJ export noise first. Rules apply to Bulk Fix queries, Fix Metadata auto-search from filename, and WAV → FLAC tag embedding when the classic pattern does not match.

  1. Strip a trailing _PN suffix (Platinum Notes), case-insensitive.
  2. If the stem matches [slot] - [BPM] - [artist] - [title] (hyphen-separated Ableton-style), use artist and title from that pattern.
  3. Otherwise strip Rekordbox-style noise: trailing Camelot key + BPM (e.g. 2A 120), then a leading slot prefix (e.g. A02 ).
Example stem (no extension) Search / tag string
A06 - 139 - Members Of Mayday - 10 In 01 Artist + title from the four-part pattern
A02 Christian Loeffler All Comes (Mind Against Remix) 2A 120 Christian Loeffler All Comes (Mind Against Remix)
A02 Ripperton Unfold 2A 119 Ripperton Unfold

Metadata sources

Source Typical use
Discogs Release metadata, artwork, tracklists; API token recommended
Bandcamp Track/album pages; search via public HTML
Apple Music iTunes Search API + catalogue pages
Spotify Metadata and artwork when fetching from Spotify URLs
SoundCloud Track pages — embedded hydration JSON (/user/slug track URLs)
Beatport Track pages — Next.js __NEXT_DATA__ (/track/… URLs)
Generic URL Open Graph title/image where available

Fix search API: GET /api/search?q=… merges Apple + Discogs + Bandcamp. Optional source=apple_music|discogs|bandcamp and limit restrict to one catalogue (used by Fix Metadata fallback UI). For SoundCloud or Beatport, paste full track URLs into Fetch from URL / Bulk Fix rows — those hosts are not in the combined text search endpoint.

Supported file roles

Role Formats
Extract & Normalise output FLAC, MP3, AAC (M4A) per Settings
Fix & Inspect FLAC (Vorbis + pictures), MP3 (ID3v2 + APIC), M4A/AAC/MP4 (iTunes atoms), OGG (Vorbis comments + METADATA_BLOCK_PICTURE)
Inspect artwork resize FLAC only (Rekordbox-friendly dimensions)

Unicode: Scraped metadata is handled as UTF-8 so accented characters survive into tags.

Metadata source URL in files

The app stores the fetching URL in a dedicated tag field (e.g. DJMETAMANAGER_SOURCE_URL in Vorbis comments / MP3 TXXX). Older files may still show a legacy key name; the reader accepts both for compatibility.

Processing log

JSON log on disk records extract/fix activity including metadata_source_url, metadata_source_type, kind (extract vs fix), profile, and loudness targets when relevant. Use it to re-tag after external renames (e.g. Platinum Notes) via the workflows described in the README.

Batch re-tag API

POST /api/retag-batch applies log-derived metadata to many files in a folder. Details and request shape are in app.py and the README.

Selected HTTP routes (automation integrators)

The UI is a thin client over Flask JSON endpoints. Examples:

  • GET /api/browse, POST /api/source-recording/rename, POST /api/source-recording/delete, GET /api/browse-audio, GET /api/browse-wav, GET /api/browse-folders — list paths / manage source recordings
  • POST /api/probe, POST /api/analyse — audio metrics
  • POST /api/extract — extract from video
  • POST /api/fetch-metadata — full metadata from URL; optional track hints
  • POST /api/retag — save one file from Fix
  • POST /api/normalise / POST /api/normalise-flac — loudness encode
  • POST /api/convert-wav-to-flac, POST /api/convert-wav-bulk — WAV batching
  • GET /api/bulk-fix/scan, POST /api/bulk-fix/scan-paths, POST /api/bulk-fix/suggest, POST /api/bulk-fix/apply

Full table: see repository README.

UI state in the browser

djmm.pageState in localStorage holds debounced drafts for Extract, Fix, Inspect, Normalise, and Settings. Settings clears its draft after a successful save so the next load matches the server. WAV → FLAC and Bulk Fix use additional keys (batch offsets, djmm.bulkFixHandoff, etc.). Nothing is synced to the cloud by the stock app.

Branches

Feature development may track branch v2 before merging to master. Verify which build your guide documents when distributing binaries or PDF exports.
