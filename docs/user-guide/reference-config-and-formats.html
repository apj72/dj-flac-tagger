Reference

Configuration keys, filename normalisation rules, supported formats, metadata sources, processing log concepts, selected HTTP API routes, and browser-side UI state. For the canonical technical spec, the repository README.md and source remain authoritative.

Configuration file (config.json)

Start from config.json.example. Typical keys include source_dir, destination_dir, extract_profile (flac, mp3, or aac/m4a), platinum_notes_app, pn_output_suffix, target_lufs, target_true_peak, loudness_verify_enabled, extract_mkv_audio_analysis_enabled (Extract tab: run full loudness analysis when selecting .mkv recordings), and fix_retain_filename_suffixes (Fix / Bulk Fix: strip configured trailing fragments from the filename stem for search, and keep them when renaming to Artist - Title). Positive LUFS input (e.g. 11.5) is interpreted as negative integrated loudness where the app supports that shorthand. From a bundled macOS .app, the live file sits under ~/Library/Application Support/DJ MetaManager/config.json; see README (macOS app bundle section).

Filename search and tags

The app does not search on raw stems. It normalises DJ export noise first. Rules apply to Bulk Fix queries, Fix Metadata auto-search from filename, and WAV → FLAC tag embedding when the classic pattern does not match.

Configurable trailing suffixes (Fix Metadata & Bulk Fix)

In Settings, Fix Metadata — filename suffixes to strip for search, keep when renaming defines an ordered list of rules (one per line in the UI). Each line is either a literal string matched at the end of the filename stem (before the extension), or a line starting with regex: followed by a regular expression (Python on the server; JavaScript in the Fix tab). A regex: pattern should usually end with $ so only a true suffix matches—for example three alphanumeric characters in parentheses: regex:_bpm\([A-Za-z0-9]{3}\)$.

The server repeatedly peels matching fragments from the end of the stem (longest-first within each pass follows the rule order you configured). The peeled fragments are not used in the online search query. If you turn on rename to Artist - Title, those fragments are appended back after the title part and before the file extension (e.g. Artist - Title_warped.flac). This is useful for custom markers such as _warped after Ableton or other offline processing. If normalised extract is enabled on Extract, loudness is still measured on the server during extract regardless of this setting.

  1. Strip a trailing _PN suffix (Platinum Notes), case-insensitive.
  2. If the stem matches [slot] - [BPM] - [artist] - [title] (hyphen-separated Ableton-style), use artist and title from that pattern.
  3. Otherwise strip Rekordbox-style noise: trailing Camelot key + BPM (e.g. 2A 120), then a leading slot prefix (e.g. A02 ).

A06 - 139 - Members Of Mayday - 10 In 01 Artist + title from the four-part pattern
A02 Christian Loeffler All Comes (Mind Against Remix) 2A 120 Christian Loeffler All Comes (Mind Against Remix)

Metadata sources

Source Typical use
Discogs Release metadata, artwork, tracklists; API token recommended
Bandcamp Track/album pages; search via public HTML
Apple Music iTunes Search API + catalogue pages
Spotify Metadata and artwork when fetching from Spotify URLs
SoundCloud Track pages — embedded hydration JSON (/user/slug track URLs)
Beatport Track pages — Next.js __NEXT_DATA__ (/track/… URLs)
Generic URL Open Graph title/image where available

Fix search API: GET /api/search?q=… merges Apple + Discogs + Bandcamp and can include SoundCloud hits in the combined result set. Optional source=apple_music|discogs|bandcamp|soundcloud and limit restrict to one catalogue (used by Fix Metadata fallback UI). Beatport is typically used by pasting full track URLs into Fetch from URL / Bulk Fix rows.

Supported file roles

Role Formats
Extract & Normalise output FLAC, MP3, AAC (M4A) per Settings
Fix & Inspect FLAC (Vorbis + pictures), MP3 (ID3v2 + APIC), M4A/AAC/MP4 (iTunes atoms), OGG (Vorbis comments + METADATA_BLOCK_PICTURE)
Inspect artwork resize FLAC only (Rekordbox-friendly dimensions)

Unicode: Scraped metadata is handled as UTF-8 so accented characters survive into tags.

Metadata source URL in files

The app stores the fetching URL in a dedicated tag field (e.g. DJMETAMANAGER_SOURCE_URL in Vorbis comments / MP3 TXXX). Older files may still show a legacy key name; the reader accepts both for compatibility.

Processing log

JSON log on disk records extract/fix activity including metadata_source_url, metadata_source_type, kind (extract vs fix), profile, and loudness targets when relevant. Use it to re-tag after external renames (e.g. Platinum Notes) via the workflows described in the README. From the .app bundle, the live file defaults to ~/Library/Application Support/DJ MetaManager/processing_log.json.

Batch re-tag API

POST /api/retag-batch applies log-derived metadata to many files in a folder. Details and request shape are in app.py and the README.

Selected HTTP routes (automation integrators)

The UI is a thin client over Flask JSON endpoints. Examples:

  • GET /api/browse, POST /api/source-recording/rename, POST /api/source-recording/delete, GET /api/browse-audio, GET /api/browse-wav, GET /api/browse-folders — list paths / manage source recordings
  • POST /api/probe, POST /api/analyse — audio metrics
  • POST /api/extract — extract from video
  • POST /api/fetch-metadata — full metadata from URL; optional track hints
  • POST /api/retag — save one file from Fix (tags, optional artwork, optional rename)
  • POST /api/retag-artwork — embed cover only; JSON filepath plus artwork_base64/artwork_mime or artwork_url
  • GET /api/stream-audio?path=… — stream local audio for the in-browser preview bar (Range-capable)
  • POST /api/normalise / POST /api/normalise-flac — loudness encode
  • POST /api/convert-wav-to-flac, POST /api/convert-wav-bulk — WAV batching
  • GET /api/bulk-fix/scan, POST /api/bulk-fix/scan-paths, POST /api/bulk-fix/suggest, POST /api/bulk-fix/apply

Full table: see repository README.

UI state in the browser

djmm.pageState in localStorage holds debounced drafts for Extract, Fix, Inspect, Normalise, and Settings. Settings clears its draft after a successful save so the next load matches the server. WAV → FLAC and Bulk Fix use additional keys (batch offsets, djmm.bulkFixHandoff, etc.). The audio preview bar stores djmmPlayerVolume separately.

Theme and visuals (not written to config.json): djmm.themePreference — dark, light, or system; djmm.pageBackgroundEnabled — 1 or 0 toggles full-page scenic art (settings UI: “Show page background image”). Implemented in static/theme-init.js for early apply. Repo art lives under design/UI/ (dark_background.png, light_background.png, header wordmarks); the app ships copies under static/ for HTTP.

Nothing is synced to the cloud by the stock app.
